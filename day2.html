Explain six ways to display JS code;

1-Embedded within HTML: JavaScript code can be directly embedded within HTML
script tags. This is a common way to include JavaScript within web pages.
2-External JavaScript File:
You can also write JavaScript code in a separate file with a .js extension and include it in your HTML using the
script tag with the src attribute pointing to the file.
3-Using the Browser Console:
You can directly type and execute JavaScript code in the
browser's developer console, which is useful for debugging and testing.
4-Server-Side JavaScript:
JavaScript code can be executed on the server using platforms like Node.js. Server-side JavaScript files typically have a
.js extension and are executed by the Node.js runtime.
5-Asynchronous Module Definition (AMD):
AMD is a specification for modular code in JavaScript.
Libraries like RequireJS implement AMD and allow you to define modules and load them asynchronously.
6-Using JavaScript Frameworks and Libraries:
Modern JavaScript frameworks like React, Angular, and Vue.js provide their own ways to structure and display
JavaScript code within components or templates. These are some common ways to display JavaScript code, each suited to different use cases and scenarios.

Java:
Strongly Typed and Static:
Java is strongly typed, meaning that variable types must be explicitly declared and adhered to throughout the program.
It is also static, which means that type checking occurs at compile time.

Object-Oriented:
Java follows the object-oriented programming (OOP) paradigm, emphasizing the use of objects and classes.
It provides features like inheritance, encapsulation, and polymorphism.

Platform Independence:
Java’s Virtual Machine (VM) allows compiled programs to run anywhere, regardless of the platform.
This concept is known as “Write Once, Run Anywhere” (WORA).

Robust and Secure:
Java’s strict type system and memory management contribute to its robustness.
It includes features like garbage collection and exception handling for better security.

Used for Standalone Applications:
Java creates standalone applications that can run independently.
Java applets (small applications) are fully contained programs that can be embedded in web pages.


JavaScript:
Weakly Typed and Dynamic:
JavaScript is weakly typed, allowing more flexibility in variable types.
It is dynamic, with type checking occurring during runtime.

Primarily Used for Web Development:
JavaScript is mainly used for web development.
It runs in web browsers and interacts with HTML and CSS to create dynamic web pages.

Event-Driven and Asynchronous:
JavaScript is event-driven, responding to user actions like clicks and form submissions.
It supports asynchronous programming using callbacks, promises, and async/await.

Client-Side Scripting:
JavaScript executes on the client side (browser).
It enhances user experience by enabling interactivity and real-time updates.

Libraries and Frameworks:
JavaScript has a rich ecosystem of libraries (e.g., React, Angular, Vue) and frameworks (e.g., Node.js) for efficient development.
